Sistema de Gestión de Garage
Decisiones de Diseño

Durante el desarrollo del proyecto se buscó mantener una estructura simple, clara y organizada, aplicando los conceptos vistos en clase.

Se utilizó una clase abstracta llamada Vehiculo para representar las características comunes de todos los vehículos, evitando repetir código en las subclases.

Las clases Moto, Auto y Camion heredan de Vehiculo y cada una implementa su propio cálculo de costo y cantidad de espacios ocupados. De esta manera se aplicó herencia y polimorfismo.

También se utilizaron interfaces para definir comportamientos comunes del sistema:
- Calculable → para el cálculo de costos
- Mostrable → para mostrar información de los vehículos

Para almacenar los vehículos se utilizó un ArrayList, ya que permite agregar, eliminar y recorrer elementos de forma sencilla.

El sistema incluye validaciones para evitar:
- patentes duplicadas
- horas inválidas
- ingresos sin espacio disponible
- retiro de vehículos inexistentes

Los errores se manejan mediante excepciones personalizadas y bloques try-catch para evitar que el programa finalice inesperadamente.

Se decidió implementar una interfaz por consola mediante un menú interactivo para facilitar el uso del sistema y permitir probar todas sus funcionalidades de forma simple.

El proyecto fue desarrollado de manera incremental utilizando Git y GitHub para llevar control de versiones y registrar los avances realizados.
